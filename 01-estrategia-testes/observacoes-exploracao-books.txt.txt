Exploração inicial da API Books - FakeRESTApi

Base URL:
https://fakerestapi.azurewebsites.net

Swagger:
https://fakerestapi.azurewebsites.net/index.html

Endpoint principal:
Books - /api/v1/Books

Resultados observados:

1. GET /api/v1/Books
Resultado: 200 OK
Observação: Retorna lista de livros com os campos id, title, description, pageCount, excerpt e publishDate.

2. GET /api/v1/Books/1
Resultado: 200 OK
Observação: Retorna livro existente com id 1.

3. GET /api/v1/Books/999999
Resultado: 404 Not Found
Observação: Comportamento esperado para ID inexistente.

4. POST /api/v1/Books
Payload testado com id 501.
Resultado: 200 OK
Observação: Retorna o objeto enviado no response body.

5. GET /api/v1/Books/501 após POST
Resultado: 404 Not Found
Observação: O livro criado via POST não foi persistido.

6. PUT /api/v1/Books/1
Resultado: 200 OK
Observação: Retorna o objeto atualizado no response body.

7. GET /api/v1/Books/1 após PUT
Resultado: 200 OK
Observação: Retorna o dado original Book 1. A atualização não foi persistida.

8. DELETE /api/v1/Books/1
Resultado: 200 OK
Observação: Retorna sucesso sem body.

9. GET /api/v1/Books/1 após DELETE
Resultado: 200 OK
Observação: O livro continua sendo retornado. A exclusão não foi persistida.

Conclusão:
A API Books permite simular operações CRUD, porém as operações POST, PUT e DELETE não persistem alteração de estado. Esse comportamento deve ser documentado como limitação do ambiente FakeRESTApi e considerado na estratégia de testes, no plano de testes e na automação.
